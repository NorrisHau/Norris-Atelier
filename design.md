# Finance Dive 设计系统与布局规范（CSS Grid）

## 1. 设计目标

- **现代**：卡片化、层次化、柔和阴影和统一圆角。
- **直观**：主页信息路径清晰，首屏即可看懂“这是一个什么博客，下一步看什么”。
- **简洁**：减少视觉噪音，统一间距和组件语义。
- **可扩展**：通过 Grid 区域命名和组件命名，支持后续快速新增模块。

---

## 2. 响应式布局策略

### 2.1 断点

- `mobile`：`< 640px`
- `tablet`：`640px - 1023px`
- `desktop`：`>= 1024px`

### 2.2 首页 Grid（`.home-shell`）

- 移动端：单列流式布局。
- 桌面端：双列布局，主内容 + 侧边阅读路径。

桌面网格区域：

```txt
"hero side"
"panel panel"
"cta cta"
```

---

## 3. 视觉语言与设计标准

### 3.1 颜色与主题

- 使用 CSS 变量驱动明暗主题（`--background`, `--foreground`, `--accent`, `--card-bg` 等）。
- 卡片背景使用 `color-mix` 与透明层叠，避免纯白硬边。

### 3.2 圆角、边框与阴影

- 主容器和模块建议圆角：`rounded-2xl` ~ `rounded-3xl`。
- 边框统一使用 `border-border` 体系。
- 阴影层级：
  - 常规卡片：`--shadow-soft`
  - 悬浮态可进一步抬升（如文章卡）。

### 3.3 字体层级

- 标题（品牌、主标题）：衬线风格（`Charter / Source Serif Pro / Noto Serif SC`）。
- 正文、元信息、按钮：无衬线（`Inter / Noto Sans SC`）。

### 3.4 间距体系

- 组件内边距优先 `p-5 / p-6 / p-7`。
- 模块垂直间距优先 `gap-4 / gap-6`。
- 全站主容器采用 `app-layout` 控制最大宽度与横向边距。

---

## 4. 关键组件规范

### 4.1 顶部导航（Header）

- `site-header-grid`：两列结构（品牌 + 导航）。
- `app-header`：粘性定位 + 毛玻璃背景 + 细分隔线。
- 导航在移动端保留折叠菜单交互；桌面端水平排列。

### 4.2 首页 Hero（`.home-hero`）

- 包含：眉题（eyebrow）/ 大标题 / 副标题 / 主题切换 chips。
- 背景：轻度径向高光，强调品牌和首屏聚焦。

### 4.3 阅读路径侧栏（`.home-side-panel`）

- 内容：建议阅读顺序 + 章节路径。
- 桌面端 sticky，跟随滚动但保持顶部留白（`top: 6.5rem`）。

### 4.4 主题内容区（`.home-panel`）

- 采用统一卡片容器规范（圆角、边框、背景、阴影）。
- 每个主题面板都应具备标题、简介、列表。

### 4.5 文章卡片（`.bento-card`）

- 采用 Grid：
  - 移动端单列（文本优先）。
  - `>=640px` 双列（内容区 + 缩略图）。
- 悬浮态：边框强调 + 轻微上浮 + 标题下划线动画。

### 4.6 底部（Footer）

- `site-footer-grid` 使用圆角顶边和轻背景层。
- 社交链接 + 版权信息保持双端可读。

---

## 5. 开发约定（给后续开发者/Agent）

1. **新增页面优先使用 Grid**，避免用大量 `flex + 嵌套` 模拟栅格。
2. **保持语义化命名**：
   - 布局层：`home-shell`, `home-panel`, `site-header-grid`。
   - 组件层：`xxx__title`, `xxx__desc`, `xxx__list`。
3. **优先复用现有设计 token**，不要直接硬编码新颜色。
4. **移动优先**：先写单列，再通过 `@media (min-width: 1024px)` 扩展桌面布局。
5. **交互一致性**：
   - 可点击元素保持统一 focus 样式；
   - 动画时长控制在 `180ms ~ 360ms` 范围。
6. **可访问性要求**：
   - 所有交互控件具备 `aria-*`；
   - 面板切换保持 `tablist / tab / tabpanel` 语义。

---

## 6. 后续扩展建议

- 可在桌面端加入“最近更新 / 热门标签”二级侧栏，但需与当前 `home-side-panel` 共用卡片规范。
- 可将文章列表拆分为“精选 + 最新”两个 Grid 区块，避免列表过长带来的阅读疲劳。
- 可在 `design.md` 旁新增 `tokens.md`，沉淀尺寸、阴影、动画、z-index 的统一表。
